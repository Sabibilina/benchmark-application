| Trade-off | Cost benefit | Risk | Validation |
| --- | --- | --- | --- |
| Short Prometheus retention for smoke/calibration, longer retention for full benchmark. | Reduces disk and memory during cheap runs. | Short runs may lose historical debugging data. | Check retention settings per profile and confirm benchmark evidence is exported before teardown. |
| Smaller streaming dummy segment payloads in smoke/calibration. | Reduces local network and CPU cost. | Understates full network payload pressure. | Document payload size per profile and run full payload only in benchmark profiles. |
| Benchmark-only infrastructure exporters. | Reduces idle containers in normal runtime. | Default runtime has less infrastructure detail. | Confirm exporter targets are present in benchmark profile and absent only by design in low-cost runtime. |
| Bounded retry/circuit breaker behavior. | Prevents cascading failure and wasted retries. | May surface failures earlier under transient downstream slowness. | Focused integration tests and latency/error metrics under downstream failure scenarios. |
| Gateway-only external routing. | Reduces host-port sprawl and supports replicas. | Gateway can become a bottleneck. | Gateway metrics and k6 calibration before scaling application services further. |
