  Query logic:
  - must clause: multi_match if q is non-empty; match_all otherwise.
  - filter clauses: each provided filter appended independently; all are ANDed

  id, trackId, title, artist, album, genre, year, bpm (= tempo), popularity,
  durationMs, danceability, energy, explicit, country, label

  ---
  Data Population Strategy

  SearchIndexSeeder (implements ApplicationRunner):

  1. If search.seed.enabled is false, skip.
  2. Count documents in the songs index. If count > 0, skip (idempotency).
  3. Create the songs index with the mapping above if it does not exist.
  4. Open data/songs.csv from classpath (same CSV bundled in the JAR — same
  approach as D-037).
  5. Parse with Apache Commons CSV; build SongDocument objects.
  6. Bulk-index in batches of search.seed.batch-size (default 500) using the
  opensearch-java BulkRequest.
  7. Use trackId as the OpenSearch document _id for upsert safety on re-seed.
  8. Log row count and elapsed time on completion.

  The CSV is a copy of the same file already at
  services/catalog-service/src/main/resources/data/songs.csv (85k rows, 19
  columns).
