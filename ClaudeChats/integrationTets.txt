   so the next test can use it without re-creating fixtures.                    
                                                                                
  ---                                                                           
                                                                                
  ## Rules                                                                      
                                                                                
  1. Every test that calls a protected endpoint must send `Authorization:       
  Bearer <token>`.                                                              
  2. Tests verify HTTP status codes first, then response body fields.           
  3. Do not hardcode song IDs — fetch a real song ID from `GET /catalog/songs`  
  in `@BeforeAll`.                                                              
  4. When verifying cross-service async flows (history, charts,                 
  recommendations, notifications), always use Awaitility:                       
     ```java                                                                    
     await().atMost(Duration.ofSeconds(20)).pollInterval(Duration.ofSeconds(1)) 
  .untilAsserted(() -> {                                                        
         given().header("Authorization", "Bearer " + token)                     
             .when().get(ServiceUrls.ANALYTICS + "/analytics/me/history")       
             .then().statusCode(200).body("size()", greaterThan(0));            
     });                                                                        
     ```                                                                        
  5. Use unique user credentials per test class (UUID-based) so tests do not    
  interfere with each other.                                                    
  6. `FullUserJourneyIT` uses a single static user for the whole class — state  
  flows from one `@Order` test to the next.

  Prerequisites: Docker Compose stack fully up and all 8 services healthy.      
                                                                                
  ```bash                                                                       
  cd e2e-tests                                                                  
  mvn test                                                                      
  ```                                                                           
                                                                                
  To run a single flow:                                                         
  ```bash                                                                       
  mvn test -Dtest=PlaylistFlowIT                                                
  mvn test -Dtest=FullUserJourneyIT                                             
  ```                                                                           
                                                                                
  A test failure in `PlaylistFlowIT` or `HistoryFlowIT` directly pinpoints      
  which backend service has a bug without needing to read logs manually.
