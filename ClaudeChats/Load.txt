❯ how do i know if scalability worked? Only from the load generator and tests?  

⏺ No — the load generator and tests tell you what the system did, not how it 
  scaled to do it. You need multiple signal layers to confirm scaling actually
  worked:

  1. Load test results (k6)
  The k6 thresholds tell you whether SLOs held under load (p99 latency, error
  rate). But passing thresholds doesn't prove scaling worked — a single
  over-provisioned instance could pass too.

  2. Prometheus / Grafana metrics
  This is where you confirm scaling specifically:
  - Per-instance request rate — are all replicas receiving traffic, or is one
  doing everything?
  - CPU and memory per pod — are they staying in a healthy range as load
  increases, rather than one instance saturating?
  - JVM heap and GC pause time — signals whether instances are under real
  pressure
  - Nginx upstream connection distribution — confirms the load balancer is
  actually spreading load

  3. Kafka consumer lag
  For analytics-service and recommendation-service, the key signal is whether
  lag on playback-events stays bounded during the 40K eps steady-state phase.
  Growing lag means those services can't keep up regardless of HTTP layer
  health.

  4. Database connection pool
  PgBouncer or HikariCP pool exhaustion is often the first thing that breaks
  under scale. Pool wait time spiking means the bottleneck is DB connections,
  not compute.

  5. The scaling behavior itself
  If you're using Docker Swarm or Kubernetes replicas, you should observe:
  - New instances starting during the warm-up phase
  - Traffic redistributing across them (visible in per-instance request counts
  in Grafana)
  - Instances staying up without OOMKill or crash-restart loops

  The full answer is: k6 tells you the outcome (did users get fast responses?),
  Grafana tells you the mechanism (did the system actually distribute load
  across multiple instances?). You need both to say scaling worked. The Grafana
  scaling dashboard you already have is the right place to look for the
  mechanism side.
